# 周报结构模板 + Wording 生成规则

> 本文件定义周报输出的标准结构和 Wording 生成规则。

---

```markdown

> 数据截止：{YYYY-MM-DD} | 分析日期：{今天日期} | 模式：{新建/刷新}

## 时间进度基准
（Q1时间进度、当月时间进度）

### 拉新GMV Top3 及 SMROI 趋势

## 模块2：素养项目组周营收（团队口径）

## 模块3：各项目组GMV进度及SMROI达标（B标）
### Q1季度进度
### 当月进度

## ⚡ 与上周对比（复利跟踪）
（画像卡存在时生成，标注🆕/❗/✅/📈/📉）

（按 🔴紧急 / 🟡重要 排列）

```

## 二、Wording 生成规则

### 整体 Wording

- 数据来源：`周营收-财务口径` Sheet 合计行
- 环比颜色：正增长🟢，负增长🔴
- Top3 项目组：按总营收**绝对变动额**排序（本周-上周），取前3个正向贡献最大的
- 绝对变动计算：本周总营收 - 上周总营收（上周值 = 本周值 ÷ (1+环比%)）
- `[*]` 标记：需用户手动填写定性分析，AI 仅填充定量数据
- 如本周总体下降，改为"本周下降主要受以下项目组影响"，列出负向贡献最大的 3 个

### 素养 Wording

- 数据来源：`周营收-团队口径` Sheet 中素养类项目组
- 项目组固定顺序：小图灵、KP、Deepthink、纵横工作室
- 小图灵和 KP 写环比和 `[*]` 说明；Deepthink 和纵横工作室只写数值

## 三、格式强制要求（必须严格遵守，不可简化）

以下格式要求是强制性的，**每次生成报告前必须对照检查**，不得以任何理由简化或省略：

1. **模块1 — 必须用表格**：总营收/拉新GMV/续报GMV/SMROI 四行数据用 Markdown 表格，含环比列；拉新Top3 和续报Top3 均用表格，不得改为文字列表
2. **模块2 — 必须逐项目组列表格**：所有素养项目组（小图灵/KP/Deepthink/纵横工作室/博闻/思维/线下店）必须每行一个，包含总营收/市场GMV/辅导GMV/市场SMROI/环比/趋势六列，不得合并或省略
3. **模块3 — 必须逐项目组列表格**：Q1进度表和3月进度表必须包含全部12个项目组，每个项目组一行，不得只列合计行或分组汇总行
4. **复利跟踪 — 必须用对比表格**：上周vs本周风险等级对比表，每个项目组一行，含变化标注列
5. **课程业务（LCT）分析 — 必须包含**：本周速览表格、分产品线分析、关键洞察，不得省略课程业务部分
6. **Wording — 必须用代码块**：整体Wording和素养Wording均用 ``` 代码块包裹，保持可直接复制格式

**自检指令**：生成报告后，在写入文件前，逐项确认以上6点是否满足。如有任何一项不满足，重新生成对应模块。

**参考标准**：`examples/W9-report-format-reference.md` 是格式基准文件，每次生成时必须对照该文件的结构和信息密度进行自检。
